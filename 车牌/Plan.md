# 上海车牌拍卖价格预测研究计划

## TL;DR

> **目标**：基于历史上海车牌拍卖数据，使用Stata构建OLS预测模型，预测拍卖当天成交价格
> 
> **交付物**：
> - 数据清洗 + 描述性统计 + OLS回归建模 + 样本外预测 + 最终报告
> - 所有do文件、回归表格(outreg2)、预测结果(markdown)、log文件、Report.md
>
> **最终输出**：将本计划内容写入项目根目录下的 `Plan.md`

---

## 一、项目概述

**目标**：基于历史数据，使用OLS回归构建上海车牌拍卖当天11:00时的价格预测模型（最低成交价/平均成交价），用于辅助实际拍卖出价决策。

**工具**：Stata（命令：`stata -b do <dofile>.do`）

**数据**：`shanghai_car_license_auction_2010_2026.csv`（2010年1月至2026年2月，共194条月度数据）

**训练/测试集划分**：2025年10月之前为训练集（191条），2025年10月及之后为测试集（5条：2025年10-12月、2026年1-2月）

## 二、数据概况

### 2.1 原始变量

| 变量名 | 含义 | 类型 | 备注 |
|--------|------|------|------|
| 拍卖时间 | 如"2010年1月" | 字符串 | 需解析为年份和月份 |
| 投放数量 | 当月投放牌照数 | 数值 | |
| 最低成交价 | 当月最低成交价（元） | 数值 | 预测目标之一 |
| 平均成交价 | 当月平均成交价（元） | 数值 | 预测目标之一 |
| 最低成交价截止时间 | 如"11:29:34第9位" | 字符串 | 可拆分为时间和位次 |
| 投标人数 | 当月参与竞拍人数 | 数值 | |
| 警示价 | 官方警示价格（元） | 数值 | 2014年前为空值 |

### 2.2 已发现的数据问题

1. **2010年12月异常值**：最低成交价10,400元、平均成交价15,970元，远低于前后月份（约40,000-45,000元），疑似数据错误或特殊政策事件，需查证
2. **警示价缺失**：2010-2013年警示价列为空，需考虑如何处理（警示价制度约于2014年引入）
3. **最低成交价截止时间**格式不一致：部分记录时间在10:56-10:59（首次出价阶段），大部分在11:29（修改出价最后阶段）
4. **中文编码**：拍卖时间为中文格式，需在Stata中正确解析

## 三、研究步骤

### 步骤1：数据清洗与变量构建

**do文件**：`01_data_cleaning.do`

具体操作：
1. 导入CSV数据，处理中文编码问题
2. 从"拍卖时间"中提取**年份（year）**和**月份（month）**数值变量
3. 生成**时间序列标识**：`time = (year - 2010) * 12 + month`，用于时间趋势
4. 从"最低成交价截止时间"中提取**截止时间的分钟数**和**位次排名**（如有分析价值）
5. 计算**派生变量**：
   - `bid_ratio = 投标人数 / 投放数量`（竞拍比，即人车比）
   - `price_spread = 平均成交价 - 最低成交价`（价差）
   - `ln_min_price = ln(最低成交价)`（对数变换）
   - `ln_avg_price = ln(平均成交价)`
   - `ln_bidders = ln(投标人数)`
   - `ln_supply = ln(投放数量)`
   - `ln_warning_price = ln(警示价)`（仅2014年后可用）
6. 生成**滞后变量**：
   - `L1_min_price`, `L2_min_price`（滞后1期、2期最低成交价）
   - `L1_avg_price`, `L2_avg_price`
   - `L1_bidders`, `L1_supply`
   - 对数版本的对应滞后项
7. 生成**月份虚拟变量**（`i.month`）
8. 处理异常值：标记或剔除2010年12月数据，说明理由
9. 标记训练集与测试集：`gen train = (time <= 189)` (2025年9月为第189期)
10. 保存清洗后的dta文件

**注意事项**：
- 在拍卖当天11:00，已知信息包括：本期的投放数量、警示价、投标人数（11:00可见）、以及所有历史数据。因此模型中可以使用**当期**的投放数量、警示价、投标人数作为解释变量
- 不可使用当期的成交价（因为那是要预测的目标）

### 步骤2：描述性统计

**do文件**：`02_descriptive_stats.do`

具体操作：
1. 全样本主要变量的**描述性统计表**（均值、标准差、最大值、最小值、中位数）：
   - 投放数量、最低成交价、平均成交价、投标人数、警示价、竞拍比
2. **分年份的描述性统计**或趋势：按年份计算各变量均值
3. **时间趋势图**（使用`twoway`命令）：
   - 最低成交价和平均成交价的时间走势
   - 投放数量的时间走势
   - 投标人数的时间走势
   - 竞拍比（人车比）的时间走势
   - 警示价与成交价的对比走势（2014年后）
4. **相关系数矩阵**：主要数值变量之间的相关系数
5. 将图表导出为png，统计表导出为txt/csv
6. 初步排查数据错漏：
   - 检查是否有缺失值
   - 检查各变量范围是否合理
   - 特别关注2010年12月异常值
   - 检查是否有重复月份

### 步骤3：OLS回归建模

**do文件**：`03_ols_models.do`

需要尝试以下模型规格（使用训练集数据，即2025年10月之前）：

#### A. 基准模型（水平值）— 预测最低成交价

| 模型编号 | 被解释变量 | 解释变量 | 说明 |
|----------|-----------|---------|------|
| M1 | 最低成交价 | 投放数量, 投标人数, 警示价 | 基本信息 |
| M2 | 最低成交价 | M1 + 时间趋势(time) | 加入线性时间趋势 |
| M3 | 最低成交价 | M2 + i.month | 加入月份固定效应 |
| M4 | 最低成交价 | M3 + L1_min_price | 加入滞后被解释变量 |
| M5 | 最低成交价 | M4 + L2_min_price | 加入二阶滞后 |
| M6 | 最低成交价 | M4 + bid_ratio | 加入竞拍比 |
| M7 | 最低成交价 | M4 + 多项式项(time^2, bidders^2) | 加入多项式 |
| M8 | 最低成交价 | M4 + 交乘项(supply×bidders等) | 加入交互项 |

#### B. 对数模型 — 预测最低成交价

| 模型编号 | 被解释变量 | 解释变量 | 说明 |
|----------|-----------|---------|------|
| M9 | ln(最低成交价) | ln(投放数量), ln(投标人数), ln(警示价) | 对数基准 |
| M10 | ln(最低成交价) | M9 + time | 加入时间趋势 |
| M11 | ln(最低成交价) | M10 + i.month | 加入月份固定效应 |
| M12 | ln(最低成交价) | M11 + L1.ln_min_price | 加入滞后项 |
| M13 | ln(最低成交价) | M12 + 多项式/交乘项 | 扩展模型 |

#### C. 平均成交价模型

对上述模型框架复制一遍，将被解释变量改为**平均成交价**（及其对数形式），模型编号M14-M26。

#### D. 特殊处理

- **警示价缺失的样本**：考虑两种策略：(a) 仅使用2014年后有警示价的子样本；(b) 全样本但不包含警示价变量
- **对数变换的回调调整**：对于对数模型，预测时需要进行Duan (1983) smearing估计或使用 $\hat{y} = \exp(\hat{\ln y} + \hat{\sigma}^2/2)$ 进行调整，因为 $E[\hat{\ln y}|x] \neq \ln E[y|x]$

#### 回归结果导出

所有回归表格使用 `outreg2` 导出为txt格式：
```stata
outreg2 using "results/reg_table_1.txt", replace ctitle(M1) 
```

### 步骤4：样本外预测与评估

**do文件**：`04_prediction.do`

具体操作：
1. 对每个模型，使用训练集估计的系数，在测试集上生成预测值
2. 对于对数模型，需要进行回调调整：
   - 计算训练集残差的方差 $\hat{\sigma}^2$
   - 调整预测值：$\hat{y} = \exp(\hat{\ln y} + \hat{\sigma}^2 / 2)$
3. 计算样本外预测指标：
   - **MAE**（平均绝对误差）
   - **RMSE**（均方根误差）
   - **MAPE**（平均绝对百分比误差）
   - 各期预测误差明细
4. 比较最低成交价 vs 平均成交价作为预测目标的精度
5. 将预测结果整理为markdown格式表格输出

### 步骤5：模型选择与最终报告

**do文件**：`05_model_selection.do`

1. 汇总所有模型的样本内拟合指标（R²、调整R²、AIC/BIC）和样本外预测指标
2. 选择最优模型，给出理由
3. 使用最优模型对未来（如2026年3月）进行预测示例

**报告**：`Report.md`

报告结构：
1. **研究背景与目的**
2. **数据描述与清洗**
   - 描述性统计表格
   - 关键变量的时间趋势图
   - 数据异常处理说明
3. **模型设定与回归结果**
   - 所有回归表格（从outreg2输出嵌入）
   - 各模型的经济学解释与显著性分析
   - 对数模型的回调调整说明
4. **样本外预测结果**
   - 各模型预测误差对比表（markdown格式）
   - 最低成交价 vs 平均成交价预测精度对比
5. **最终模型选择与评估**
   - 最优模型及其依据
   - 模型局限性讨论
   - 实际应用建议（如何根据预测价格制定出价策略）

## 六、关键注意事项

1. **变量可用性约束**：在实际预测场景中（拍卖日11:00），可用变量为：投放数量、警示价、投标人数（11:00后可见）、以及所有历史数据（含滞后值）。模型中**不能**使用当期的成交价作为解释变量。

2. **对数回调问题**：若使用 ln(y) 作为被解释变量，预测时 exp(E[ln y|x]) 会系统性低估 E[y|x]。需使用Duan's smearing estimate或正态性假设下的 exp(μ̂ + σ̂²/2) 进行校正。

3. **警示价处理**：
   - 2014年前无警示价数据，若模型包含警示价，训练样本仅约130条（2014年1月至2025年9月）
   - 可同时尝试含/不含警示价的模型，比较预测精度

4. **异常值处理**：2010年12月数据需特别关注（最低成交价10,400元，仅为前后月份的1/4），需判断是否为数据录入错误或真实事件（如政策变动导致的价格暴跌），并决定是否剔除。

5. **outreg2安装**：确保Stata已安装outreg2包（`ssc install outreg2`）

6. **Log文件**：在master.do开头使用 `log using "log/master.log", replace text` 保留完整运行记录。

7. **时间序列注意**：加入滞后项后需注意样本量减少，以及潜在的序列相关问题（可用Durbin-Watson或Breusch-Godfrey检验辅助判断）。
